Long on the Great Wall

The goal of this project was to develop a game in C++ with OpenGL using Blender as modeling tool and, separately, render a complex scene related to the game in Blender.
The game is an endless runner in wich a Chinese dragon flies over the Great Wall trying to collect coins while avoiding obstacles.
The rendering represents a scene in a Chinese restaurant and joins techniques such as particles, water simulation, Cycles materials, etc.

Remote control of Windows machines

The goal of this project was to develop a client-server solution for remote control of Windows machines.
The client part had to be able to connect to many server machines and, for each, get the list of open windows and, if needed, send key combinations either to all the servers running a specific process or to the window currently on focus.
The server part had to listen for an incoming connection from the client, send all the events concerning its own windows (such as on focus, closed, open) and deliver the key combos sent by the client.
The client was developed in C# with WPF for the graphic user interface, the server was developed in C++, and the two communicated through Json.

Enigma

The goal of this project was to develop a game in Unity3D, with Blender as modeling tool, and to render part of the scene with stereoscopic rendering.
The game has a first-person perspective and leads the player into a small open world that joins some of the most popular paintings by Giorgio De Chirico, such as The Seer, The Song of Love, The Uncertainty of the Poet, etc.
The goal of the game is to solve puzzles hidden among the buildings and the other elements of the paintings.
Also, some paintings were reproduced and rendered stereoscopically with Blender.

All of this is required to be developed with Unity3D, and all its limitations. There are a lot of precision and efficiency issues, that I'm solving through the use of high precision data type, floating origin, scaled space, and many other techniques.
